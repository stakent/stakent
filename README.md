# Dariusz Walat

Trustworthy output from untrustworthy data. I build systems that catch what's silently wrong — contradictory records, a garbled scan, a confidently-wrong model — so it surfaces as a flag today, not a six-month catastrophe.

I use LLMs where they earn their place (development, meaning, embeddings, plain-language access to jargon-locked data) and keep them out of decisions and the data flow, which stay deterministic and verifiable. An LLM can help you find the answer; it can't be the answer.

Senior Software Engineer, 20+ years — from industrial control systems to data pipelines. Python, Go, PostgreSQL.

Currently working on: turning messy Polish legal PDFs into a correctness-checked graph — the hard part is the verification, not the AI.

Everything here is personal work, built in free time.

I write about where AI coding agents actually fail and why verification is the real work: [walat.eu](https://walat.eu)
